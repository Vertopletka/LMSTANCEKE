1. Общие сведения
Название проекта: Tancheke. Язык программирования: Python 3.x. Основная графическая библиотека: Arcade. Жанр: Arcade / Top-down shooter / Grid-based puzzle.

2. Цель проекта
Разработка программного продукта, представляющего собой танковый симулятор с сеточной системой позиционирования, элементами разрушаемого окружения и прогрессирующей сложностью уровней.

3. Игровые механики и функциональные требования
3.1. Управление и перемещение
    Игровое поле разделено на логические ячейки (тайлы) размером 40x40 пикселей.
    Перемещение всех динамических объектов (игрок, враги, ящики) строго привязано к сетке.
    Танк игрока должен поддерживать плавную интерполяцию движения между центрами ячеек.
    Реализована механика взаимодействия с объектами: возможность передвигать ящики (Push-механика) при условии отсутствия препятствий за ними.

3.2. Боевая система
    Стрельба осуществляется по прямой линии в направлении текущего поворота танка.
    Ограничение скорострельности реализовано через кулдаун (параметр SHOOT_COOLDOWN).
    Снаряды уничтожаются при столкновении со стенами или разрушаемыми объектами.

3.3. Вражеские юниты и ИИ
    Рядовые противники: Передвигаются случайным образом по свободным ячейкам. Ведут автоматический огонь при нахождении игрока на одной линии видимости по осям X или Y.
    Босс (Секретный уровень): Стационарный или малоподвижный объект с повышенным запасом прочности (10 единиц HP). Атакует «веерной» стрельбой (несколько снарядов под разными углами).

3.4. Система прогрессии и уровней
    Наличие как минимум трех стандартных уровней, генерируемых на основе карты проходимости и случайного размещения объектов.
    Условие перехода на следующий уровень: полное уничтожение всех вражеских юнитов на текущей карте.
    Наличие скрытого уровня, активируемого по специальному коду.

4. Графический интерфейс и визуальные эффекты
    HUD (Heads-Up Display): Отображение текущего счета, количества жизней и номера уровня в верхней части экрана.
    Система частиц: Генерация визуальных эффектов взрыва при уничтожении танков или разрушении ящиков.
    Floating Text: Вывод всплывающих текстовых уведомлений (например, при получении бонусов).
    Динамическая камера: Поддержка функции масштабирования (Zoom) с фокусом на танке игрока.

5. Технические требования к хранению данных
    Рекорды: Сохранение максимального счета в текстовый файл record.txt.
    Чтение данных: При запуске приложение должно считывать файл рекорда и отображать его в главном меню.
    Сброс: Предусмотреть функцию удаления файла или обнуления значения рекорда.

6. Ресурсы
    Графика: Спрайты в формате PNG (танки, стены, ящики).
    Звук: Звуковые файлы в формате MP3 для выстрелов и подбора бонусов.

7. Условия завершения игры
    Победа: Прохождение всех уровней или уничтожение босса.
    Поражение: Обнуление счетчика жизней игрока (Lives <= 0).
